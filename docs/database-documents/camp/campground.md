# campground
* environment_tags
    定義營區可使用的環境標籤，例如高海拔、有雲海。
* facility_tags
    定義營區可使用的設施標籤，例如獨立衛浴、裝備租借。
* campground_environment_tags
    營區與環境標籤的多對多關聯表。
* campground_facility_tags
    營區與設施標籤的多對多關聯表。



## 關聯與資料流
campgrounds
├─ N:M environment_tags
│     └─ 透過 campground_environment_tags
└─ N:M facility_tags
      └─ 透過 campground_facility_tags

### 關聯
* environment_tags：環境標籤的值池；同一個標籤可套用至多個營區。
* facility_tags：設施標籤的值池；同一個標籤可套用至多個營區。
* campground_environment_tags：以營區與環境標籤的複合主鍵，避免同一營區重複套用相同環境標籤。
* campground_facility_tags：以營區與設施標籤的複合主鍵，避免同一營區重複套用相同設施標籤。
* 刪除營區時，兩張關聯表會同步刪除關聯資料（`ON DELETE CASCADE`）；已被關聯使用的標籤不可直接刪除（`ON DELETE RESTRICT`）。

### 資料流程
營區標籤建立後：
1. 先在 environment_tags 或 facility_tags 建立標籤代碼、顯示名稱與排序。
2. 將標籤套用到營區時，在對應的 campground_*_tags 關聯表新增一筆資料。
3. 前台取得營區資料後，將各營區的標籤用於篩選與卡片／詳情顯示。
4. 標籤不再提供選擇時，應設為 `active = false`，保留既有營區關聯與歷史資料。



## 欄位說明
### environment_tags
* id                    環境標籤流水號，BIGINT
                        GENERATED BY DEFAULT AS IDENTITY

* code                  標籤代碼，VARCHAR(64)，UNIQUE。
* label                 顯示給使用者的標籤名稱，VARCHAR(100)，UNIQUE。

* sort_order            顯示順序，預設 0，必填，必須大於等於 0。
                        *idx_environment_tags_active_sort* (active、sort_order)

* active                是否仍提供選擇，預設 true

### facility_tags
* id                    設施標籤流水號，BIGINT
                        GENERATED BY DEFAULT AS IDENTITY

* code                  標籤代碼，VARCHAR(64)，UNIQUE。
* label                 顯示給使用者的標籤名稱，VARCHAR(100)，UNIQUE。
* sort_order            顯示順序，預設 0，必須大於等於 0
                        *idx_facility_tags_active_sort* (active、sort_order)

* active                是否仍提供選擇，預設 true

### campground_environment_tags
* campground_id         營區識別碼，VARCHAR(32)
                        與 tag_id 組成複合主鍵 `pk_campground_environment_tags`。

* tag_id                環境標籤流水號，BIGINT
                        *idx_campground_environment_tags_tag* (tag_id)
                        與 campground_id 組成複合主鍵。

### campground_facility_tags
* campground_id         營區識別碼，VARCHAR(32)
                        與 tag_id 組成複合主鍵 `pk_campground_facility_tags`。

* tag_id                設施標籤流水號，BIGINT
                        *idx_campground_facility_tags_tag* (tag_id)
                        與 campground_id 組成複合主鍵。



## 運作模式
* 環境標籤主檔 > environment_tags
* 設施標籤主檔 > facility_tags
* 營區環境標籤套用 > campground_environment_tags
* 營區設施標籤套用 > campground_facility_tags
* 停用標籤應更新 `active = false`，而非直接刪除；schema 未提供 `created_at`、`updated_at` 與軟刪除欄位。



## 程式碼追蹤
* 前台營區篩選
    `booking/js/camp-search.js`
                ↓
    `window.BookingAPI.getCampgrounds()`
                ↓
    `js/booking-api.js`
                ↓
    讀取 `data/catalog/campgrounds.json`
                ↓
    依 `environmentTags`、`facilityTags` 篩選營區

    * 目前實際執行時：
        - 不直接查詢 PostgreSQL 的四張標籤資料表。
        - 標籤值來自 `campgrounds.json` 內的 `environmentTags`、`facilityTags` 陣列。
        - 正式後端應由兩張主檔與兩張關聯表聚合回前端所需的標籤陣列。



## 可能的問題
* 高風險：前端仍讀取 JSON 標籤陣列，與正規化後的四張資料表是兩套資料來源；更新其中一端可能造成篩選結果不一致。
* 中風險：標籤主檔沒有 `created_at`、`updated_at` 與建立／修改者，無法追蹤設定異動。
* 中風險：`code` 與 `label` 僅限制 UNIQUE，未限制不可為空白字串；後端或資料庫應補上空白檢查。
* 低風險：標籤停用後，資料庫仍允許新增關聯至 `active = false` 的標籤；後端應拒絕這類新增操作。
