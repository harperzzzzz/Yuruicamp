# calendar
* calendar_dates
    台灣日曆日期、假日標記、假日名稱與資料版本。
* zone_blocks
    指定營區營位區在日期區間內停售／維修時所扣減的可售數量。
* campground_closures
    指定營區在日期區間或固定星期幾公休；命中時該營區所有營位區不可預約。



## 關聯與資料流
campgrounds
├─ 1:N campground_closures
└─ 1:N campground_zones
      └─ 1:N zone_blocks

admin_users
├─ 1:N zone_blocks
└─ 1:N campground_closures

calendar_dates
└─ 提供平日／假日價格計算依據

### 關聯
* calendar_dates：獨立的日曆主檔，不以外鍵連至營區；以 `calendar_date` 作為單日資料的主鍵。
* zone_blocks：以 `(campground_id, zone_id)` 複合外鍵連至 campground_zones，確保營位區一定屬於所填的營區；同時記錄建立者。
* campground_closures：關聯一個營區與建立者；公休規則可為日期區間或每週固定日。
* 營區、營位區與管理者被其他資料引用時皆不可直接刪除（`ON DELETE RESTRICT`）；更新識別碼會級聯更新（`ON UPDATE CASCADE`）。

### 資料流程
預約可用性與價格計算時：
1. 依入住日期讀取 calendar_dates，判斷是否為假日並選擇平日或假日價格。
2. 取得指定營區與營位區的基礎可售數量 `campground_zones.total_sites`。
3. 新增或修改 zone_blocks 時，`trg_zone_blocks_validate_capacity` 會驗證單筆及同日重疊封鎖總量不得超過 `campground_zones.total_sites`。
4. 匯總命中日期區間的 zone_blocks.blocked_quantity，從可售數量扣除；`get_zone_availability()` 以 `greatest(..., 0)` 保證可用量不為負數。
5. 檢查 campground_closures：若日期命中日期區間或每週規則，該營區全部營位區當晚皆不可預約。
6. 後台建立停售或公休規則時，必須記錄建立者與原因。



## 欄位說明
### calendar_dates
* calendar_date         日曆日期

* is_holiday            是否為假日
                        *idx_calendar_dates_holiday_date* (is_holiday、calendar_date)

* holiday_name          假日名稱，NULL
                        非假日時必須為 NULL；假日可不填名稱。

* source_version        日曆資料來源版本，不得為空白。

* effective_at          此版本日曆開始生效的時間

* updated_at            更新時間，預設 now()
                        沒有自動更新的 Trigger。

### zone_blocks
* id                    營位停售流水號
                        GENERATED BY DEFAULT AS IDENTITY

* campground_id         營區識別碼
                        與 zone_id 組成複合 FK `fk_zone_blocks_campground_id_zone_id`。
                        *idx_zone_blocks_zone_dates* 的第一欄。

* zone_id               營位區識別碼
                        與 campground_id 組成複合 FK，確保營位區屬於同一營區。
                        *idx_zone_blocks_zone_dates* (campground_id、zone_id、start_date、end_date)

* start_date            停售開始日期
* end_date              停售結束日期必須大於等於 start_date。
* blocked_quantity      封鎖的可售營位數，必須大於 0。
                        `trg_zone_blocks_validate_capacity` 會鎖定對應營位區，阻止單筆或同日重疊封鎖總量超過 total_sites。
* reason                停售原因，不得為空白

* created_by            建立者管理員識別碼
                        FK `fk_zone_blocks_created_by` → admin_users.id。
                        *idx_zone_blocks_created_by* (created_by)

* created_at            建立時間，預設 now()
* updated_at            更新時間，預設 now()
                        沒有自動更新的 Trigger。

### campground_closures
* id                    營區公休流水號
                        GENERATED BY DEFAULT AS IDENTITY

* campground_id         公休營區識別碼
                        *idx_campground_closures_campground_dates*
                        *idx_campground_closures_campground_weekday*

* closure_type          規則類型，VARCHAR(16)，只允許 `date_range` 或 `weekly`。
                        `ck_campground_closures_type`

* start_date            日期區間公休的開始日期，僅 `date_range`
* end_date              日期區間公休的結束日期，僅 `date_range` 必填且不得早於 start_date。

* weekday               每週公休日，SMALLINT，僅 `weekly` ，範圍為 0–6。
                        *idx_campground_closures_campground_weekday* (campground_id、weekday)

* effective_from        每週規則的生效開始日期，僅 `weekly` 必填。
* effective_to          每週規則的生效結束日期，僅 `weekly` 必填且不得早於 effective_from。
* reason                公休原因，TEXT，必填，不得為空白。

* created_by            建立者管理員識別碼
                        FK `fk_campground_closures_created_by` → admin_users.id。
                        *idx_campground_closures_created_by* (created_by)

* created_at            建立時間，預設 now()，必填。

* updated_at            更新時間，預設 now()，必填。
                        沒有自動更新的 Trigger。

* `ck_campground_closures_payload` 會強制兩種規則欄位互斥：
    - `date_range`：start_date、end_date 必填；weekday、effective_from、effective_to 必須為 NULL。
    - `weekly`：weekday、effective_from、effective_to 必填；start_date、end_date 必須為 NULL。


## 運作模式
* 平假日資料與價格依據 > calendar_dates
* 指定營位區維修／停售 > zone_blocks
* 指定營區整體公休 > campground_closures
* zone_blocks 只扣減指定營位區的可售數；campground_closures 則關閉該營區的所有營位區。
* `validate_zone_block_capacity()` 與 `trg_zone_blocks_validate_capacity` 負責封鎖容量驗證；`get_zone_availability()` 負責逐日彙總預約、封鎖與公休結果。



## 程式碼追蹤
* 前台可用性查詢
    `js/booking-api.js`
                ↓
    `BookingAPI.getZoneBlocks()`、`BookingAPI.getClosures()`
                ↓
    讀取 `data/admin/zone-blocks.json`、`data/admin/campground-closures.json`
                ↓
    `js/booking-availability.js`
                ↓
    以停售數量與公休規則計算可預約營位

    * 目前實際執行時：
        - 不直接讀取 PostgreSQL 的 calendar_dates、zone_blocks、campground_closures。
        - zone blocks 使用 JSON；closures 會額外合併 localStorage overlay。
        - 前台 `booking/js/camp-detail.js` 目前以週五、週六判定假日，未讀取 calendar_dates。



## 可能的問題
* 高風險：前台假日判定固定為週五、週六，未使用 calendar_dates；國定假日與日曆資料庫的價格可能不一致。
* 高風險：停售與公休在前台使用 JSON／localStorage，但正式資料庫有三張對應表，兩套資料來源可能造成可用性不一致。
* 中風險：updated_at 僅有預設值，後續更新不會由資料庫自動更新；應由後端服務統一處理。

### 舊資料位置
* 舊版來源檔：`data/admin/zone-blocks.json`、`data/admin/campground-closures.json`。
* 現行 `public.zone_blocks.id` 與 `public.campground_closures.id` 是唯一的正式識別碼，不保存舊資料 ID。
