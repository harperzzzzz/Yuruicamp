# inventory-movements
    庫存異動表頭；記錄庫存領域、異動類型、來源／目的地點、負責員工與過帳狀態。
* store_inventory_movement_items  
    商城庫存異動明細關聯表；記錄商城商品規格及異動當下的 SKU、品名快照。
* rental_inventory_movement_items  
    租借庫存異動明細關聯表；記錄租借 SKU 規格及異動當下的 SKU、品名快照。


## 關聯與資料流
inventory_movements
  ├─ store_inventory_movement_items
  └─ rental_inventory_movement_items

### 關聯
* inventory_movements：異動單表頭，為商城與租借共用
* store_inventory_movement_items：商城異動明細；透過inventory_domain 驗證
* rental_inventory_movement_items：租借異動明細；透過inventory_domain 驗證

### 資料流程
建立庫存異動時：
1. 在 `inventory_movements` 建立 `draft` 表頭，指定庫存領域、異動類型、來源或目的地點、員工與原因。
2. 商城異動明細寫入 `store_inventory_movement_items`；租借異動明細寫入 `rental_inventory_movement_items`。
3. 明細的 `inventory_domain` 必須與表頭相同，資料庫以複合外鍵強制驗證。
4. 異動類型決定地點組合：進貨只允許目的地、損耗只允許來源地、移轉必須同時有不同的來源與目的地。
5. 表頭改為 `posted` 時必須填入 `posted_at`；`draft` 與 `cancelled` 不得填入 `posted_at`。
6. 讀取清單或明細時，使用 `inventory_movement_items_view` 或 `inventory_movement_dto_view` 將兩個領域的資料統一呈現。


## 欄位說明
### inventory_movements
* id                            異動流水號
                                `GENERATED BY DEFAULT AS IDENTITY`。

* movement_no                   異動單號，UNIQUE。
* inventory_domain              庫存領域，只允許 `store` 或 `rental`。

* movement_type                 異動類型，不可為空白。
                                `receipt`
                                `write_off`
                                `transfer`
                                `conversion_out`
                                `conversion_in`

* status                        異動狀態，只允許 `draft`、`posted`、`cancelled`。

* source_location_id            來源庫存地點，NULL；
                                與 `inventory_domain` 組合後外鍵指向 `inventory_locations`。 (location 指向地點，domain 驗證store / rental)

* destination_location_id       目的庫存地點，NULL；
                                與 `inventory_domain` 組合後外鍵指向 `inventory_locations`。

* employee_id                   負責員工，外鍵指向 `admin_users.id`。
                                *idx_inventory_movements_employee ON (employee_id)*

* reason                        異動原因，不可為空白。
* occurred_at                   異動發生時間，由呼叫端提供。
* posted_at                     過帳時間，NULL；只有 `status = posted` 時必填。
* created_at                    建立時間，預設 `now()`。
* updated_at                    最後更新時間，預設 `now()`；DDL 未定義自動更新 Trigger。
*UNIQUE(id, inventory_domain)*
*idx_inventory_movements_domain_status_time ON (inventory_domain、status、occurred_at)*
*idx_inventory_movements_source_domain ON (source_location_id、inventory_domain)*
*idx_inventory_movements_destination_domain ON (destination_location_id、inventory_domain)*

### store_inventory_movement_items
* id                    明細流水號
                        `GENERATED BY DEFAULT AS IDENTITY`

* movement_id           異動表頭識別碼；
                        與 `inventory_domain` 複合外鍵指向 `inventory_movements`。(驗證是否為store)

* inventory_domain      固定為 `store`，預設 `store`。

* variant_id            商城商品規格識別碼，外鍵指向 `product_variants.id`。
                        *idx_store_inventory_movement_items_variant ON (variant_id)*

* sku_snapshot          異動當下 SKU 快照
* item_name_snapshot    異動當下品名快照
* quantity              異動數量，必須大於 `0`。
*idx_store_inventory_movement_items_movement ON (movement_id、inventory_domain)*

### rental_inventory_movement_items
* id                    明細流水號
                        `GENERATED BY DEFAULT AS IDENTITY`。

* movement_id           異動表頭識別碼；
                        與 `inventory_domain` 複合外鍵指向 `inventory_movements`。(驗證是否為rental)

* inventory_domain      固定為 `rental`，預設 `rental`。

* rental_sku_variant_id 租借 SKU 規格識別碼
                        *idx_rental_inventory_movement_items_variant ON (rental_sku_variant_id)*

* sku_snapshot          異動當下 SKU 快照
* item_name_snapshot    異動當下品名快照
* quantity              異動數量，必須大於 `0`。
*idx_rental_inventory_movement_items_movement ON (movement_id、inventory_domain)*



## 運作模式
* 異動單表頭 > `inventory_movements`
* 商城異動明細 > `store_inventory_movement_items`
* 租借異動明細 > `rental_inventory_movement_items`
* 統一明細讀取 > `inventory_movement_items_view`
* 後台／報表 JSON DTO > `inventory_movement_dto_view`

### 領域與狀態
* 一張異動單不能同時包含商城與租借明細。
* 明細中的 SKU 與品名是異動快照，不應在商品主檔更名後回寫。

## 可能的問題
* 高風險：前端異動記錄目前讀寫 JSON 與記憶體快取，尚未寫入 `inventory_movements`、`store_inventory_movement_items` 或 `rental_inventory_movement_items`。
* 高風險：現行前端 mock 資料仍可含 `進貨`、`損耗`、`移轉` 等中文類型；正式 API 串接前必須改為送出 canonical 值
* 高風險：未完成後端規則前，資料庫層無法阻止事後修改。

* 高風險: inventory_movements.status 的 `posted` 與 `cancelled` 異動的不可變性，以及明細僅能在草稿階段編輯
        目前 `schema_copy.sql` 中沒有這些 Trigger。移交後端執行

* 中風險：前端異動資料格式使用顯示用欄位（例如 `fromStore`、`toStore`、`productName`）
        正式表的 location ID、領域、規格 ID、快照欄位不同，串接前需建立 DTO 轉換與驗證。

* 中風險：三張表的 `updated_at` 只有預設值，更新時不會自動刷新。
* 低風險：正式異動表不再保留舊 JSON ID，不應在正式交易流程使用。
