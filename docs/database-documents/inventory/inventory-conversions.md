# inventory-conversions
    商城轉租借的配對紀錄
* inventory_movements  
    轉換的異動表頭；商城端使用 `conversion_out`，租借端使用 `conversion_in`。
* inventory_locations  
    轉換來源與目的庫存地點；商城與租借地點以 `inventory_domain` 隔離。
* store_inventory_movement_items  
    一張商城異動單可有多筆商品變體明細。
* rental_inventory_movement_items  
    一張租借異動單可有多筆租借異動明細。


## 關聯與資料流
```text
product_variants
└─ 1:N inventory_conversions.source_variant_id

rental_sku_variants
└─ 1:N inventory_conversions.destination_rental_variant_id

inventory_locations
├─ 1:N inventory_conversions.source_location_id      （商城地點）
└─ 1:N inventory_conversions.destination_location_id （租借地點）

inventory_movements
├─ 1:1 inventory_conversions.source_movement_id      （store / conversion_out）
└─ 1:1 inventory_conversions.destination_movement_id （rental / conversion_in）

inventory_movements
├─ 1:N store_inventory_movement_items
└─ 1:N rental_inventory_movement_items
```

### 關聯
* inventory_conversions：跨領域關聯交易；來源是商城商品規格與商城地點，目的為租借 SKU 規格與租借地點。
* source_movement_id：商城端異動表頭，應為 `inventory_domain = store`、`movement_type = conversion_out`。
* destination_movement_id：租借端異動表頭，應為 `inventory_domain = rental`、`movement_type = conversion_in`。
* store_inventory_movement_items：商城端表頭的明細，連到 `product_variants`。
* rental_inventory_movement_items：租借端表頭的明細，連到 `rental_sku_variants`。
* inventory_locations：轉換表直接保存來源與目的地點，並由異動表頭的複合外鍵確認異動單與地點的領域相容。

### 資料流程
商城規格轉為租借規格時：
1. 建立商城領域的 `inventory_movements` 草稿，類型為 `conversion_out`，填入 `source_location_id`。
2. 在 `store_inventory_movement_items` 建立商城規格與數量的轉出明細。
3. 建立租借領域的 `inventory_movements` 草稿，類型為 `conversion_in`，填入 `destination_location_id`。
4. 在 `rental_inventory_movement_items` 建立租借 SKU 規格與數量的轉入明細。
5. 在 `inventory_conversions` 寫入兩張異動表頭、兩端規格、兩端地點、數量與 `idempotency_key`。
6. 兩張異動單過帳後，商城現有庫存扣減、租借現有庫存增加；轉換紀錄提供兩端的稽核連結。


## 欄位說明
### inventory_locations
* id                    庫存地點識別碼，由呼叫端提供。
* code                  庫存地點代碼，NIQUE。
* inventory_domain      庫存類型，只允許 `store` 或 `rental`。

* type                  分類至store/rental 後的狀態
                        `store` 只允許 `main`、`branch`、`inspection`、`repair`、`damaged`。
                        `rental` 只允許 `main`、`campground`、`inspection`、`repair`、`damaged`。

* branch_id             所屬門市，NULL；只有 `type = branch` 且 `inventory_domain = store` 時必填
                        `type` 非 `branch` 時必須為 NULL。
                        *idx_inventory_locations_branch ON (branch_id)*

* name                  庫位顯示名稱
* active                是否啟用，預設 `true`。
* created_at            建立時間，預設 `now()`。
* updated_at            最後更新時間，預設 `now()`；DDL 未定義自動更新 Trigger。

*CHECK：庫存領域、地點類型，以及門市類型與 `branch_id` 的相容性。*
*idx_inventory_locations_domain_type_active ON (inventory_domain、type、active)*

### inventory_conversions
* id                              轉換流水號
                                 `GENERATED BY DEFAULT AS IDENTITY`

* source_movement_id              商城轉出異動表頭
* destination_movement_id         租借轉入異動表頭
                                  *idx_inventory_conversions_destination_location ON (destination_location_id)*

* source_variant_id               商城商品規格
                                  *idx_inventory_conversions_source_variant ON (source_variant_id)*

* destination_rental_variant_id   租借 SKU 規格
                                  *idx_inventory_conversions_destination_variant ON (destination_rental_variant_id)*

* source_location_id              商城來源地點
                                  *idx_inventory_conversions_source_location ON (source_location_id)*

* destination_location_id         租借目的地點
                                  *idx_inventory_conversions_destination_movement ON (destination_movement_id)*
                                  
* quantity                        轉換數量，必須大於 `0`。
* idempotency_key                 唯一轉換編號，UNIQUE；同一請求不可建立第二筆轉換。
* created_at                      建立時間，預設 `now()`。
*UNIQUE：(source_movement_id, destination_movement_id)*


### inventory_movements
* id                              異動流水號
                                  `GENERATED BY DEFAULT AS IDENTITY`。

* movement_no                     異動單號，UNIQUE。
* inventory_domain                庫存領域，只允許 `store`、`rental`。
* movement_type                   異動類型；
                                  標準化類型：`receipt`、`write_off`、`transfer`、`conversion_out`、`conversion_in`。

* status                          狀態，只允許 `draft`、`posted`、`cancelled`。
* source_location_id              來源地點，
                                  與 `inventory_domain` 以複合外鍵指向 `inventory_locations(id, inventory_domain)`。 (location 紀錄地點，domain 驗證)

* destination_location_id         目的地點，
                                  與 `inventory_domain` 以複合外鍵指向 `inventory_locations(id, inventory_domain)`。

* employee_id                     負責員工，外鍵指向 `admin_users.id`。
* reason                          異動原因，不可為空白。
* occurred_at                     異動發生時間。
* posted_at                       過帳時間；`posted` 時必填，`draft`、`cancelled` 時必須為 NULL。
* created_at／updated_at          建立與更新時間；皆預設 `now()`，DDL 未定義更新時自動刷新 Trigger。

*UNIQUE：`movement_no`、`(id, inventory_domain)`*

* movement_type 會驗證來源／目的地是否符合異動類型:
movement_type	來源庫位	目的庫位	庫存效果
---------
receipt	        無	        有	       入庫
write_off	有	        無	       損耗／報廢出庫
transfer	有	        有，且不可相同	一端減、一端加
conversion_out	商城庫位	 無	       商城轉出
conversion_in	無	        租借庫位       租借轉入


### store_inventory_movement_items
* id                              明細流水號
                                  `GENERATED BY DEFAULT AS IDENTITY`

* movement_id                     異動表頭；
                                  與 `inventory_domain` 複合外鍵指向 `inventory_movements`。

* inventory_domain                固定為 `store`
* variant_id                      商城商品規格，外鍵指向 `product_variants.id`。
* sku_snapshot                    異動時的 SKU 快照。
* item_name_snapshot              異動時的品名快照。
* quantity                        明細數量，必須大於 `0`。
*idx_store_inventory_movement_items_movement ON (movement_id、inventory_domain)*


### rental_inventory_movement_items
* id                              明細流水號
                                  `GENERATED BY DEFAULT AS IDENTITY`。

* movement_id                     異動表頭；
                                  與 `inventory_domain` 複合外鍵指向 `inventory_movements`。

* inventory_domain                固定為 `rental`
* rental_sku_variant_id           租借 SKU 規格
* sku_snapshot                    異動時的 SKU 快照。
* item_name_snapshot              異動時的品名快照。
* quantity                        明細數量，必須大於 `0`。
*idx_rental_inventory_movement_items_movement ON (movement_id、inventory_domain)*



## 運作模式
* 商城規格轉出 > `inventory_movements` + `store_inventory_movement_items`
* 租借規格轉入 > `inventory_movements` + `rental_inventory_movement_items`
* 成對轉換關聯與冪等防重 > `inventory_conversions`
* 來源／目的地點與領域驗證 > `inventory_locations`
* 跨領域明細讀取 > `inventory_movement_items_view`

## 可能的問題

* 高風險：`inventory_conversions` 的外鍵只確認兩端異動、規格與地點存在；
        「來源必為商城轉出、目的必為租借轉入、兩端明細與數量相符」等跨表業務驗證移交後端，但目前尚未有對應實作。

* 高風險：前端調撥只改 JSON／記憶體快取；正式 API 未完成前，不會在 PostgreSQL 建立可稽核的成對轉換資料。
* 中風險：`inventory_conversions` 沒有 `updated_at`；建立後若允許修改，無法記錄最後更新時間。實務上轉換完成後應視為不可修改。
* 中風險：來源與目的異動表頭各自可包含多筆明細；應由服務層驗證轉換列指定的規格、地點與數量確實對應兩端明細。
