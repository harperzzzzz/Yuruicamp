# customers
* preference_options 
    定義可選擇的偏好項目
* customer_preferences 
    顧客與偏好項目的多對多關聯
* customer_shipping_addresses 
    保存顧客的多組收件地址
* customer_tags 
    顧客後臺自訂義標籤
* customer_tag_assignments 
    顧客後臺自訂義標籤關聯表


## 關聯與資料流
customers
├─ 1:N customer_shipping_addresses
├─ N:M preference_options
│     └─ 透過 customer_preferences
└─ N:M customer_tags
      └─ 透過 customer_tag_assignments
### 關聯
* customers：會員資料的中心。
* customer_shipping_addresses：一位會員可以有多筆地址。
* customer_preferences：會員與偏好選項的多對多關聯表。
* customer_tag_assignments：會員與標籤的多對多關聯表。
### 資料流程
會員建立後：
1. 在 customers 建立基本身分資料。
2. customers.id 未提供時，由資料庫自動產生 32 字元 UUID
3. 收件地址應寫入 customer_shipping_addresses
4. 前台問卷從 preference_options 讀取可用選項
5. 會員選擇結果寫入 customer_preferences
6. 會員標籤先建立於 customer_tags
7. 將標籤套用到會員時，在 customer_tag_assignments 建立關聯
---
* 會員軟刪除：
soft_delete_customer(customer_id)
        ↓
customers.active = false
customers.deleted_at = now()
customers.updated_at = now()
        ↓
會員不再出現在 active_customers
        ↓
保留地址、偏好、標籤、訂單及其他關聯



## 欄位說明
### customers
* id                    識別碼，預設由資料庫產生 32 字元 UUID
* name                  姓名
  phone                 電話，NULL，沒有手機格式限制

* email                 電子郵件，UNIQUE
                        *idx_customers_active_email*

  birthday              生日，NULL，沒有限制不得為未來日期
* registered_at         註冊時間，呼叫端必須自行提供。
  tier                  等級代碼，NULL。
  tier_name             等級顯示名稱，NULL。
* points                會員點數，預設 0，CHECK points >= 0。
* first_purchase_used   是否已使用首購優惠，預設 false。

* auth_provider         OAuth 登入來源
                        只允許 google、facebook、line。
                        *idx_customers_auth_provider*

  firebase_uid          Firebase Auth UID，可空。
                        首次成功 OAuth 綁定後由後端寫入。
                        *uq_customers_firebase_uid*（NULLS DISTINCT）
                        不出現在 `active_customers` View。

* created_at            建立時間，預設 now()。

* updated_at            更新時間，預設 now()
                        沒有統一自動更新。
    
  avatar_url            頭像網址，NULL

  deleted_at            軟刪除時間，NULL
                        登入與一般查詢必須同時檢查 active = true及 deleted_at IS NULL。

* status                會員狀態，預設 active，
                        (active、suspended、deleted)
*資料庫禁止直接 DELETE customers*

### preference_options
* id                選項流水號
                    GENERATED BY DEFAULT AS IDENTITY

* type              偏好分類，只允許 style 或 equipment。
* code              選項代碼
* label             顯示給使用者的名稱
* sort_order        顯示順序，預設 0，必須大於等於 0
* active            是否仍提供選擇，預設 true。
* created_at        建立時間，預設 now()。

* updated_at        更新時間，預設 now()。
                    沒有自動更新的Trigger。

*(type, code) 複合 UNIQUE*
*(type, label) 複合 UNIQUE*
*idx_preference_options_type_active_sort ON (type、active、sort_order)*

### customer_preferences
* customer_id       識別碼
                    *pk_customer_preferences*

* preference_id     偏好選項流水號
                    *pk_customer_preferences*
                    *idx_customer_preferences_preference*

*(customer_id, preference_id) 是複合主鍵*

### customer_shipping_addresses
* id                地址流水號
                    GENERATED BY DEFAULT AS IDENTITY

* customer_id       地址所屬會員
                    *idx_customer_shipping_addresses_customer*
                    *idx_customer_shipping_addresses_one_default* (條件：is_default = true)

* recipient_name    收件人姓名
* postal_code       郵遞區號
* city              縣市
* district          鄉鎮市區
* address_line      街道、路段、門牌等詳細地址
* phone             收件人電話
* is_default        是否為預設地址，預設 false。
* created_at        建立時間，預設 now()。

* updated_at        更新時間，預設 now()
                    沒有自動更新的Trigger。

*每位會員最多只能有一筆 is_default = true 的地址*

### customer_tags
* id                標籤流水號
                    GENERATED BY DEFAULT AS IDENTITY。

* name              標籤名稱，UNIQUE。
* color             標籤顏色
* sort_order        顯示順序，必須大於等於 0
* active            標籤是否啟用，預設 true。
* created_at        建立時間，預設 now()。

* updated_at        更新時間，預設 now()
                    沒有自動更新的Trigger

*idx_customer_tags_active_sort (active、sort_order)*

### customer_tag_assignments
* customer_id   會員識別碼

* tag_id        標籤識別碼
                *idx_customer_tag_assignments_tag*

*同一會員不能重複套用同一個標籤* 
*(customer_id, tag_id) 是複合主鍵*
*idx_customer_tag_assignments_tag (customer_id、tag_id)*



## 運作模式
* 會員基本資料 > customers
* 偏好選項 > preference_options
* 收件地址 > customer_shipping_addresses
* 會員標籤 > customer_tags
### 軟刪除
* 會員刪除不執行 DELETE，而是更新，使用deleted_at 表示刪除時間

## 可能的問題
* 高風險：正式資料庫與前端有兩套會員資料來源
* 高風險：OAuth 登入固定使用 U001

* 高風險：customers.email 全表 UNIQUE 可能阻止軟刪除會員重新註冊
    (引入第三方登入後再解決，還要制定回復規則)
    (不給復原的話，是合理的設計)
    (現在customers.status 只有acitve, deleted, suspended 三種狀態)

* 中風險：前端 customers.json 仍有 totalSpent
    資料庫已將消費總額統一由 customer_spending_summary.total_spent 計算

* 中風險：會員等級有三套來源
    customers.tier
    customers.tier_name
    customer_spending_summary.total_spent

* 中風險：一般查詢沒有被強制使用 active_customers
    正式後端應統一：
        Repository 查詢 active_customers；或
        所有查詢固定加入有效會員條件；或
        限制應用程式資料庫帳號只能讀 View。

* 中風險：停用偏好及標籤仍會被前端組合
    前端 _loadNormalizedCustomers() 沒有過濾停用項目。
    後端 要阻止新增關聯到停用項目

* 中風險：預設地址只限制最多一筆，沒有保證至少一筆
    後端建立第一筆地址時應自動設為預設；刪除預設地址時應在同一交易中選出下一筆預設地址。

* 中風險：updated_at 不會自動更新
    用 Spring Boot Service 完成

* 低風險：registered_at 與 created_at 語意接近
    (如果兩者永遠相同，可以移除其中之一)
* 低風險：color 沒有格式限制
