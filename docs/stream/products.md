# 介面操作與後端互動

## products.html
* 此頁的意義：商城的商品總覽頁，負責顯示所有可販售商品，並提供個人化推薦、篩選、排序、分頁、規格選擇及加入購物車。
* 頁面網址：`/storefront/pages/products.html`
* 登入方式採用：Firebase＋後端會員 Session API
    - 瀏覽商品及讀取商品 API 不需要登入。
    - 共用頁首仍可讓使用者登入；Firebase 負責外部身分驗證，Session API 負責建立或同步會員資料。

### 載入時
- GET `/api/products?page=0&size=100&sort=id,asc`
    - 取得最多 100 筆可販售商品，依照id 升序排序。

### 篩選新品、熱銷品
- GET `GET /api/products?page=0&size=100&sort=createdAt,desc`
    - 取得後端依 `products.created_at` 降序排列的商品，作為新品排序順位。
    - 是否顯示新品以公開商品 `tags` 是否包含「新品」為準；開發 Seed 標記前 10 件。
- GET `GET /api/products/bestsellers?limit=100`
    - 取得帶有「熱銷」標籤的商品；開發 Seed 標記有效訂單商品數量前 6 件。
    - 回傳順序由後端依非取消、非退貨訂單的商品銷售數量計算，不需要讀取受保護的訂單 API。
    - 前端仍以公開商品 `tags` 判定是否顯示標籤。
- 三個商品請求會平行執行；其中任一請求失敗時，商品區會顯示「商品載入失敗」。

### 個人化推薦輪播
- 廣告推波會依據會員的偏好推薦商品：
- 沒有偏好或沒有符合商品時，改用商品列表隨機 12 筆作為輪播內容。

### 篩選、搜尋與排序
* 以下操作都在瀏覽器內對已載入的商品進行，不會重新呼叫後端：
    - 商品分類：單選。
    - 品牌：可複選。
    - 價格區間：最低價與最高價皆包含邊界。
    - 關鍵字搜索：比對商品名稱、品牌、分類、描述、標籤及興趣標籤。
* 支援網址參數：
    - `?filter=new`：只顯示新品。
    - `?filter=bestseller`：只顯示熱銷商品。
    - `?keyword=帳篷`：以關鍵字篩選商品。
    - 兩種參數可同時使用，例如：`products.html?filter=new&keyword=帳篷`。

### 加入購物車
- 使用者按下「加入購物車」時：
    - 前端從已載入的商品資料取得目前選取的顏色、尺寸與對應規格。
    - 購物車商品會儲存在 storage
