先讀取完所有database 文件 > 對計畫標註優先順序、階段 > 為每一階段的計畫設計一份驗證流程 > 在最後給予下一步建議和有無中途分析找到的問題

- 文件來源優先順序 :
第一順位：docs/schema.sql 的目標 DDL
第二順位：docs/database/*.md 的欄位說明
第三順位：data/** 的實際資料形狀
第四順位：docs/database-er.md 的關聯說明
第五順位：plans/** 的整合與遷移規格
第六順位：前端規格與程式實際使用方式

- 建議階段與優先順序
P0：文件與資料基線 (最高優先級)
P1：共用主檔與基礎參照
P2：商品、營區與標籤正規化
P3：租借模型

- 每階段都應使用的驗證流程
1. DDL 驗證
2. 結構驗證

4. 業務規則驗證
5. 回歸驗證
