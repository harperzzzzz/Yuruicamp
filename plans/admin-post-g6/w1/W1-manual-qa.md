# W1 本機手動點測步驟（新手版）

| 欄位 | 內容 |
|------|------|
| **對象** | Admin Post-G6 W1（01～07） |
| **前置** | 後端已起、Admin Google 白名單已寫入、前端 `USE_MOCK_API=false` |
| **資料** | `docker compose up -d` 灌入；W1 專用 ID 見下方 |

---

## 0. 先把資料庫灌好（只需一次／或要重來時）

> Docker **只有第一次建立 volume** 會跑 schema＋seed。  
> 若你之前已經 `up` 過，舊資料還在，**不會**自動重灌。

### 全新重灌（會清空本機 DB）

```powershell
cd C:\Users\Amy\Desktop\java_bootcamp\Yuruicamp
docker compose down -v
docker compose up -d
# 等 healthy 後，再跑後端 Spring Boot
```

重灌後記得：再執行一次 `021-admin-google-whitelist.example.sql`（換成你的 Google email）。

### 不砍 volume、只重跑「整包」seed

```powershell
docker exec yuruicamp-db psql -U postgres -d yuruicamp -f /docker-entrypoint-initdb.d/002-dev-seed.sql
```

若整包因本機多出來的白名單 admin email 衝突而失敗，可只重跑 W1 固定單：

```powershell
docker exec yuruicamp-db psql -U postgres -d yuruicamp -f /docker-entrypoint-initdb.d/dev/090-w1-manual-fixtures.sql
```

### W1 專用固定 ID（搜尋用）

| ID | 用途 |
|----|------|
| `W1-ORD-NOTE` | 商城訂單：測賣家備註 → 再出貨 |
| `W1-BK-NOTE` | 租借預約（含裝備）：測賣家備註（優先用這張） |
| `W1-ORD-REV` / `W1-REV-DEL` | 測評論硬刪 |
| 會員 `U001`（Amy Chen） | 測標籤／地址／偏好 |
| 商品 `P001`／規格 `V001` | 測最低庫存（main 閾值 3） |

另外還有展示用：訂單 `1`～`222`、預約 `1`～`90`、評論 `REV001`…。

---

## 1. ADM-W1-01　賣家備註（訂單＋預約）

### 訂單

1. 後台 → **訂單管理**
2. 搜尋／打開 **`W1-ORD-NOTE`**（應為已付款、待出貨）
3. 在「內部備註／賣家備註」輸入：`已電聯，明天出貨`
4. 儲存 → 應出現成功 toast；重新打開詳情，備註還在
5. **不要關掉詳情**，接著按出貨（ship）
6. 通過條件：狀態變成已出貨／後續可完成；**備註文字仍在**（備註不是狀態歷程）

### 預約（租借訂單）

1. 後台 → **預約管理**
2. 打開 **`W1-BK-NOTE`**（宜蘭礁溪、含羽絨睡袋）
3. 備註輸入：`客人要求提前入營確認`
4. 儲存 → 重開詳情備註仍在
5. 通過條件：狀態仍是 confirmed（或你沒有刻意改狀態）

---

## 2. ADM-W1-02　會員標籤池

1. 後台 → **會員管理** → 標籤池／標籤設定（依你 UI 入口）
2. **新增**標籤：名稱 `W1測試標`、選一個顏色
3. **改名**或改顏色 → 儲存成功
4. **停用**該標籤（若 UI 有）或確認「有人在用時不能硬刪」
5. 通過條件：列表看得到；Sidebar 不再顯示「未接」

> Seed 已有：`高消費`／`高退貨率`／`新會員`，也可先拿來改。

---

## 3. ADM-W1-03　標籤指派

1. 打開會員 **`U001`（Amy Chen）**
2. 編輯標籤：勾選／取消一個標籤（例如再掛 `新會員`）
3. 儲存 → 重開詳情，標籤正確
4. 回到會員列表，用 `tagId` 篩選該標籤 → 應找得到 U001
5. 通過條件：不能掛「已停用」標籤（若你剛停用過，試掛應失敗／400）

---

## 4. ADM-W1-04　預設地址

1. 打開 **`U001`** 詳情
2. 改預設地址（例如路名加 `（W1改）`）
3. 儲存 → 重開詳情地址已更新
4. 打開任意**舊訂單**（例如訂單 `1` 或 `W1-ORD-NOTE`）看收件地址 snapshot
5. 通過條件：**訂單上的收件地址沒有被改掉**（只有會員預設地址變了）

---

## 5. ADM-W1-05　偏好

1. 仍在 **`U001`**
2. 改風格／裝備偏好（例如多勾一項、拿掉一項）
3. 儲存 → 重開詳情一致
4. 通過條件：只能選 active 的偏好選項

---

## 6. ADM-W1-06　評論列表／刪除

1. 後台 → **評論管理**（不應再顯示未接）
2. 列表應有多則（含 `REV031`、`W1-REV-DEL` 等）
3. 找 **`W1-REV-DEL`**（留言開頭「【W1 專用】」）
4. 刪除 → 確認 → 列表消失
5. 通過條件：刷新後仍不在；其他評論還在  
   （若要還原：重跑 seed 即可）

---

## 7. ADM-W1-07　最低庫存

1. 後台 → **商品管理** → 打開 **`P001`（Coleman 六人帳篷）**
2. 找到規格 **`V001`**、庫位 **main**
3. Seed 已設 `minimum_quantity = 3`；改成 `5` 儲存
4. 重開商品／重新載入 → 仍是 5
5. 通過條件：改的是「最低庫存閾值」，**不是**實際 on_hand 庫存數量
