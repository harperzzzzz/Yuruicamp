# 全域共用元件 (Global Components)

## 導覽列 (Navbar)

- **一般顯示：** 左側 Logo，中央搜尋框，右側包含購物車 Icon（帶有數字 Badge）、登入/會員中心入口。
- **動態搜尋 (Search Autocomplete)：** 點擊搜尋框輸入時，下方浮現預設好的靜態下拉選單（例如列出 3-4 個 Tag 或包含關鍵字的商品名稱）。按下 Enter 或點擊搜尋按鈕，跳轉至「所有商品頁面」。
- **響應式設計 (RWD)：** 手機版 (Mobile View) 自動隱藏搜尋框與文字選單，收合為右側或左側的漢堡選單 (Hamburger Menu)。

## 全域懸浮按鈕 (Floating Action Button)

- **LINE 客服：** 固定於畫面右下角 ( position: fixed )。點擊後可開啟新分頁，模擬跳轉至 LINE 官方帳號加入好友或聊天室頁面。

# 1. 首頁 (Home)

## 主視覺 (Hero Banner)

- **滿版影片背景：** 使用露營情境短影音。HTML `<video>` 標籤需設定 autoplay, loop, muted, playsinline（確保手機端也能自動播放）。
- **行動呼籲 (CTA)：** 畫面正中央疊加高對比的「立刻探索」按鈕。

## 品牌牆輪播 (Brand Carousel)

- **呈現方式：** 在主視覺下方，放置一條橫向、無縫不間斷滾動的品牌 Logo 跑馬燈（如：Snow Peak, Coleman 等知名品牌），營造平台規模感。

## 精選商品區塊 (Product Sections)

- **最新商品 & 熱銷商品：** 分為兩個橫向區塊，各陳列 3-5 張商品卡片。
- **導流設計：** 區塊右上角或最末端放置「查看更多」按鈕，引導至對應的分類商品頁。

## 登入與註冊機制 (Modals)

- **登入視窗：** 點擊 Navbar「登入」，彈出 Modal。內含靜態「Google 登入」、「LINE 登入」按鈕。點擊後觸發 JS，將 Navbar 狀態切換為已登入（顯示會員頭像），不走後端驗證。
- **個人化問卷 (Stepper Modal)：** 登入完成後連動彈出。設計 2-3 題單選/多選（如：偏好風格、想添購的裝備），並附帶「完成」按鈕。展示時口頭說明此結果將影響首頁下方的推薦演算。

# 2. 所有商品頁面 (Product List)

## 側邊欄篩選器 (Sidebar Filter)

- **桌面版顯示：** 常駐左側，包含「商品分類」（帳篷、睡袋等）、「價格區間」（拉桿或 Input）、「品牌勾選」。
- **前端互動：** 綁定 jQuery 點擊事件，點擊特定分類或標籤時，利用 toggle class 淡入/淡出右側對應的商品卡片，展示動態篩選效果。
- **響應式設計 (RWD)：** 手機版隱藏側邊欄，畫面底部或頂部固定一顆「篩選」按鈕。點擊後以 Bottom Sheet（由下往上滑出的視窗）滿版或半版呈現篩選選單。

## 商品列表區 (Product Grid)

- **排版：** 採用 Grid 系統（如 Bootstrap 網格）排列商品卡片。
- **卡片規格：** 包含商品預覽圖、商品名稱、原價（灰色帶刪除線）、特價（顯眼亮色）、以及右下角的「加入購物車」Icon。
- **視覺微互動 (Hover Effects)：** 滑鼠懸停卡片時，整張卡片產生 Z 軸上浮陰影 (box-shadow)，或圖片執行 src 抽換（展示另一張細節圖），提高點擊誘因。

# 3. 商品描述頁面 (Product Detail)

## 上方核心區塊 (Hero Product Section)

- **左側圖集：** 上方為一張主圖，下方橫排多張小縮圖。點擊小圖時，使用 JS 動態替換主圖的 src。
- **右側購買資訊：** 包含商品名稱、價格、規格選擇（顏色/尺寸按鈕，點擊需有 Active 狀態切換）、數量增減器 (+ / - 按鈕綁定 JS 控制 Input 數值，最低為 1)。
- **免運進度條 (Progress Bar)：** 狀態寫死為 80%，文案顯示「還差 NT$ 350 即可享有免運優惠」，刺激視覺轉換。
- **行動呼籲 (CTA)：** 放置等寬或並排的大型「加入購物車」與「直接購買」按鈕。

## 加入購物車微互動 (Cart UX)

- **無跳轉回饋：** 點擊加入購物車後，不重新載入網頁。畫面右上角或中下方滑出 Toast 提示組件：「已成功加入購物車」。
- **Badge 動畫：** Navbar 上的購物車數字觸發 Bounce 或 Pulse CSS 動畫，數字動態從 0 變為 1。

## 下方詳細資訊區塊 (Tabs Section)

- **UI 結構：** 使用標籤頁 (Tabs) 切換內容，預設展開「商品介紹」。
- **商品介紹頁籤：** 滿版呈現高質感的露營情境大圖與詳細圖文規格。
- **評價區頁籤：** 靜態刻出 3-4 則買家評論，需包含：用戶頭像、五星 Icon 評分、文字回饋、以及 1-2 張買家返圖，讓版面具備真實 UGC (用戶生成內容) 的感覺。

# 4. 會員中心 (Member Center)

## 版面總體佈局 (Layout Structure)

- **桌面版顯示 (Desktop View)：** 採雙欄設計。左側配置「導覽選單 (Sidebar Menu)」，右側為「內容顯示區 (Main Content)」。
- **響應式設計 (RWD)：** 手機版將左側選單收合，改為「上方橫向滑動頁籤 (Scrollable Tabs)」或「下拉式選單 (Select/Dropdown)」，以最大化右側資料列表的視覺與操作空間。

## 4-1. 會員等級與總覽 (Profile Overview & Tiers)

- **視覺設計：** 頁面頂部（或首屏顯眼處）放置一張高質感的會員專屬「數位卡片」。
- **卡片內容：** 包含會員頭像、姓名，以及帶有戶外主題感的會員等級 Badge（如：「新手露友」、「探險家」、「營地大師」）。
- **升等進度條 (Progress Bar)：** 靜態展示升級門檻（例：「目前累積消費 NT$ 3,500 / 升級下一階需 NT$ 5,000」）。進度條填滿比例需使用 inline-style 寫死（如 width: 70%）與文案相符，展示刺激回購的營運邏輯。

## 4-2. 個人資料管理 (Profile Settings)

- **UI 結構：** 標準表單輸入框 (Input Forms)。
- **包含項目：**
  - 基本資料：姓名、電話、信箱。
  - 常用地址簿：預設收件地址設定。
  - 個人喜好設定：呼應首頁的「註冊問卷」（如：偏好野營、輕量化裝備），以 Tag 標籤或下拉選單呈現，供會員隨時修改。
- **互動回饋 (Events)：** 點擊「儲存修改」按鈕後，綁定 JS 事件阻擋表單預設提交 (e.preventDefault())，頁面不重整，直接彈出綠色 Toast 提示：「資料已更新成功」。

## 4-3. 訂單管理與查詢 (Order Management) [展示核心亮點]

- **訂單狀態頁籤 (Status Tabs)：** 頂部設計「全部、處理中、已出貨、已完成、已取消」五個頁籤。綁定 jQuery 點擊事件，動態切換下方訂單列表的顯示狀態。
- **訂單列表卡片 (Order List)：** 顯示訂單編號、下單日期、總金額、狀態 Badge、商品縮圖與「查看明細」按鈕。
- **訂單明細與 LINE 客服整合 (O2O 整合展示)：**
  - 點擊「查看明細」展開詳細購買清單與物流進度（使用 Bootstrap Modal 或 Accordion 手風琴元件實作）。
  - 明細內建「LINE 聯絡賣家」按鈕，實作時將 href 寫死帶有參數的 URI（如：[https://line.me/R/msg/text/?您好](https://line.me/R/msg/text/?您好)，我的訂單編號是#12345），手機版展示時點擊可真實喚醒 LINE 介面。
  - 桌面版配套：按鈕旁新增「顯示 QR Code」功能，點擊後彈出 QR Code 圖片，供展示時直接用手機掃描帶入上述參數網址。
- **商品評價機制 (Review Modal)：**
  - 「已完成」狀態的訂單明細內顯示「寫評價」按鈕。
  - 點擊彈出 Modal，內含 Textarea 與 5 顆評分星星（利用 CSS hover 與 jQuery 實作點擊後變更亮起狀態 active class）。

## 4-4. 折價券管理 (Coupons)

- **UI 結構：** 採用類似實體票券的卡片邊框設計（利用 CSS radial-gradient 或 mask 做出邊緣半圓形缺口），強化領取與使用感。
- **狀態切換：** 分為「可使用」、「已失效/已使用」兩個頁籤。
- **失效狀態：** 過期/已使用的票券整張套用 CSS filter: grayscale(100%) 變為灰階，並在視覺上壓上「已過期/已使用」的浮水印。
- **票券內容與互動：** 折扣名稱、有效期限、使用規則說明，以及折扣碼。點擊折扣碼旁的「複製」按鈕觸發 JS 複製功能，並顯示 Toast：「已複製」。

## 4-5. 通知總覽 (Notifications)

- **列表設計：** 條列式清單，左側以 Icon 區分分類（如：齒輪表系統通知、箱子表訂單通知、禮物表活動優惠）。
- **未讀狀態 (Unread State) 模擬：**
  - 未讀項目於左側或右側顯示小紅點，文字標題套用粗體 (font-weight: bold)。
  - 點擊該列後，透過 jQuery 移除紅點元素與粗體 class，即時模擬「已讀」的視覺行為。

# 5. 購物車與結帳頁面 (Shopping Cart & Checkout Flow)

## 5-1. 購物車面板 (Cart Offcanvas / Modal)

- **呈現方式 (UI Layout)：** 為避免打斷購物體驗，採用右側滑出的 Offcanvas 側邊欄或 Modal 呈現。內部明細區塊設定為可垂直滑動 (overflow-y: auto)。
- **空狀態設計 (Empty State)：** 購物車無商品時，顯示情境插畫（如空帳篷/空背包），並配置「去逛逛」按鈕，引導跳轉回所有商品頁面。
- **商品明細卡片：** 包含商品縮圖、名稱、單價、數量調整器 (+ / - Input)、以及「刪除」垃圾桶 Icon。
- **前端即時互動 (State Management)：**
  - 綁定 jQuery 事件，點擊 + / - 或直接輸入數量時，利用 JS 即時重新計算該列「小計」與最下方「訂單總金額」。
  - 點擊「刪除」Icon 時，觸發該列卡片的淡出動畫 (fadeOut()) 並從 DOM 移除，同時連動重算總金額。

## 5-2. 未登入跳轉邏輯 (Auth Interceptor)

- **觸發時機：** 於購物車面板點擊「前往結帳」按鈕時觸發。
- **靜態展示實作 (Mock Logic)：**
  - 宣告全域變數 let isLoggedIn = false; 模擬登入狀態。
  - 點擊「前往結帳」時進行判斷：若為 false，使用 e.preventDefault() 阻擋預設跳轉，並自動呼叫首頁設計的「登入/註冊 Modal」。
  - 若為 true（展示時可點擊假登入按鈕將變數切換為 true），則順利跳轉至「結帳頁面」。

## 5-3. 結帳頁面 (One-page Checkout)

- **版面設計 (Layout)：** 採用雙欄設計。左側為「結帳資料填寫區」，右側為固定不動的 (position: sticky)「訂單摘要與總金額 (Order Summary)」，強化購物安心感。
- **手風琴表單 (Accordion Steps)：** 左側資料區分為三個折疊面板，分段引導填寫：
  - **買家資訊 (Buyer Info)：** 姓名、電話、信箱。配置「帶入會員資料」按鈕，點擊後觸發 JS 將預設假資料自動 Value 填入 Input 中。
  - **物流配送 (Shipping)：** 提供單選按鈕 (Radio Buttons)。選項包含「宅配到府」與「實體門市取貨」。綁定 Change 事件，選擇不同物流時，右側「運費」數值動態更新，並重算最終結帳金額。
  - **付款方式 (Payment)：** 提供「信用卡/簽帳金融卡」、「LINE Pay」、「貨到付款」選項。
- **行動呼籲 (CTA)：** 頁面底部與右側摘要區下方，放置醒目的「確認結帳」滿版大按鈕。

## 5-4. 結帳成功頁面 (Order Success)

- **視覺回饋：** 點擊「確認結帳」後跳轉此頁。畫面中央顯示大型綠色 Check Icon 與「訂單成立！」祝賀字樣。
- **後續引導：** 顯示靜態的隨機訂單編號（如：#ORD-20260603），下方提供雙按鈕動線：「查看訂單明細（導向會員中心）」與「繼續購物（導回首頁）」。

# 6. 分店與合作店家 (Branches & Partners)

## 6-1. 實體分店資訊 (Store Locations)

- **版面設計 (Layout)：** 採雙欄佈局。左側為「分店列表清單」，右側為「Google Maps 靜態地圖 (iframe 嵌入)」。
- **分店卡片：** 包含門市照片、店名、營業時間、地址、聯絡電話，以及一個「規劃路線」按鈕（點擊跳轉 Google Maps 帶入目的地）。
- **互動展示 (Events)：** 綁定 jQuery 點擊事件。點擊左側某間分店卡片時，右側地圖的 iframe src 動態切換至該店的座標，且被點選的卡片加上 active class（例如左側加上顯眼的綠色邊框 Highlight）。
- **響應式設計 (RWD)：** 手機版改為單欄，地圖置頂固定或隱藏，下方呈現分店卡片列表。

## 6-2. 合作店家與營地 (Partner Network)

- **視覺佈局：** 採用網格 (Grid) 或橫向滑動輪播 (Carousel)。
- **卡片設計：** 包含合作營地/品牌的精美首圖、名稱、特色 Tag（如：#免裝備露營、#寵物友善）。
- **行銷導流：** 點擊後可跳轉至該合作店家的專屬介紹頁面，或直接彈出 Modal 顯示「專屬訂房/購買優惠碼」。

# 7. 內容行銷：部落格 (Blog & Content Marketing)

## 7-1. 文章總覽頁 (Blog Index)

- **主視覺 (Hero Article)：** 最上方放置一篇「精選文章」（如：2026 新手露營必備清單），採大圖配大標題的設計。
- **文章列表：** 下方採瀑布流 (Masonry) 或標準 3 欄 Grid 排列。每篇文章包含：封面圖、分類標籤、標題、兩行文字摘要、發布日期。

## 7-2. 文章閱讀頁 (Article Detail) [內容導購亮點]

- **閱讀體驗：** 置中單欄排版，圖文並茂，文字行距 (line-height) 設定為 1.6 - 1.8 提升閱讀舒適度。
- **文內導購模組 (Product Embedding)：** 展示商業價值的關鍵！在文章段落間，靜態插入在「所有商品頁面」設計的「商品卡片」。展示時向老師與評審說明：讀者被文章生火後，可以直接點擊加入購物車，實現內容到銷售的無縫轉換。

# 8. 客服與 QA (Customer Service)

## 8-1. QA 常見問題

- **UI 結構：** 使用手風琴 (Accordion) 折疊面板實作。分類為「購物問題」、「退換貨須知」、「會員規則」。點擊標題展開解答，保持版面乾淨。

## 8-2. 滿意度回饋問卷

- **觸發機制：** 設計為一個靜態的網頁或 Modal，假定是透過結帳後的 Email 或 LINE 推播進入。
- **表單內容：** 包含 1-10 分的 NPS (淨推薦值) 滑桿、星級評分、以及文字建議框。點擊送出後觸發 JS，隱藏表單並顯示「感謝您的回饋」Toast 與打勾動畫。
